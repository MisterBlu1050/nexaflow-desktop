# NexaAI Output Contract

> **Version:** 1.0 — April 2026  
> **Scope:** `/people-report` command (other routes: see per-route notes below)  
> **Purpose:** Define the guaranteed structure, field requirements, and data governance rules for every NexaAI output — so future developers, model swaps, and prompt updates can validate correctness without reading the full codebase.

---

## 1. Report Structure — 6 Sections

| # | Title | Source | Mandatory |
|---|-------|--------|-----------|
| 1 | Executive Summary | LLM | Yes |
| 2 | Workforce Snapshot | LLM (ground-truth figures injected) | Yes |
| 3 | Data Gaps — Classified and Remediated | LLM (dataGapMatrix from context) | Yes |
| 4 | Active HR Cases — Priority Matrix | LLM (case list from context) | Yes |
| 5 | Consolidated Action Plan | **Deterministic** (`lockedSections` in `use-command-router.ts`) | Yes |
| 6 | Scenario: Status Quo in 90 Days | **Deterministic** (`lockedSections` in `use-command-router.ts`) | Yes |

**Sections 5 and 6 are NEVER generated by the LLM.** They are pre-computed in `buildDataGapMatrix()` / action plan constants and appended by `NexaAIWindow.tsx` after the LLM response. This guarantees their content regardless of model quality or context window truncation.

---

## 2. Required Fields per Section

### Section 1 — Executive Summary
- Exactly **3 bullets**
- Each bullet format: `[Finding] → COMEX decision: [action] → P&L / roadmap consequence: [impact]`
- Bullet 3 **must** reference headcount risk linked to product velocity or revenue
- No generic statements — every finding must cite a CAS number or a named metric

### Section 2 — Workforce Snapshot
- Headcount table: Department | Headcount | % of total
- Tenure: mean + median (or explicit "Insufficient data (N=X)" if sample < 3)
- High flight-risk count (integer, from ground-truth `employees` array)
- Engineering turnover: 18% (Company Bible — hardcoded, not LLM-generated)

### Section 3 — Data Gaps
- One sub-section per entry in `DATA_GAP_MATRIX` (currently 3 gaps)
- Each gap must include a **DECISION BLOCK**:
  - Owner: named person (not a role title alone)
  - Due date: ISO format `YYYY-MM-DD`
  - KPI target: measurable outcome
  - Risk if no action: business consequence with timeline

### Section 4 — Active HR Cases
- Table: Case | Description | Severity (/5) | Legal Exposure | Immediate Action Required
- Exactly 5 rows: CAS-001 to CAS-005
- No invented cases

### Section 5 — Action Plan (deterministic)
- P&L callout blockquote (pre-written)
- Table: 7 rows (CAS-001–005 + 2 data gap actions)
- Columns: Case/Issue | Immediate Action | Owner | Due Date | KPI Target | Risk if Delayed
- All owners are **named individuals**, not generic roles

### Section 6 — Scenario (deterministic)
- Table: 3 rows (Worst / Expected / Best)
- Columns: Scenario | Trigger Event | 90-Day Outcome | Headcount Impact | Financial Risk | Legal Exposure
- Series B milestone paragraph (hardcoded figures)

## 3. Data Sources & Precedence

| Data type | Source | Override rule |
|-----------|--------|---------------|
| Department headcounts | `REAL_HEADCOUNTS` (nexaflow-context.ts) | Always wins over LLM or SQLite |
| Salary medians | `CANONICAL_MEDIANS` → SQLite → computed | In this priority order |
| Market benchmarks | `MARKET_MEDIANS` (nexaflow-context.ts) | Static; update each quarter |
| Tenure (mean/median) | Computed from `employees[].startDate` | Requires N ≥ 3; else "Insufficient data" |
| Flight risk count | Computed from `employees[]` (active + high) | Ground-truth filter, not LLM estimate |
| Active HR cases | Hardcoded CAS-001→005 | LLM must not invent additional cases |
| Data Gap Matrix | `buildDataGapMatrix()` (nexaflow-context.ts) | Factory receives runtime tenure params |

## 4. Tolerance Rules for Missing Data

| Scenario | Behaviour |
|----------|-----------|
| `employees` array empty | `flightRiskHigh = 0`; `tenureYearsArr = []`; tenure marked "Insufficient data (N=0)" |
| All `startDate` null | Same as empty array — tenure gap flagged in dataGapMatrix |
| `tenureYearsArr.length < 3` | `avgTenureYears = null`; prompt injects "Insufficient data" string, not "null" |
| Negative tenure (future startDate) | Filtered out by `v >= 0` guard in router |
| SQLite IPC unavailable | Falls back to `REAL_HEADCOUNTS` ground truth silently |
| LLM writes "SIRI" | `sanitizeContent()` in `pdfExporter.tsx` replaces with "SIRH" |
| LLM adds section 5 or "Next Steps" | `lockedSections` appended after LLM response overrides; MANDATORY STOP in prompt |

---

## 5. Forbidden LLM Behaviours

These are explicitly prohibited in the system prompt and structurally prevented where possible:

- Writing "SIRI" instead of "SIRH"
- Inventing HR cases beyond CAS-001 to CAS-005
- Mentioning offices outside Brussels HQ
- Adding a "Next Steps" section (duplicate of section 5)
- Numbering sections beyond ## 4. in its own output
- Using headcounts other than those in `REAL_HEADCOUNTS`
- Marking tenure as "Data Missing" when `tenureReliable = true`
- Fabricating salary figures not in `CANONICAL_MEDIANS` or `MARKET_MEDIANS`

## 8. Updating This Contract

When adding a new route or changing section structure:
1. Update this file first (contract = source of truth for structure)
2. Update `use-command-router.ts` system prompt to match
3. Update `lockedSections` if new deterministic sections are added
4. Run `tsc --noEmit --skipLibCheck` — 0 errors required before push
